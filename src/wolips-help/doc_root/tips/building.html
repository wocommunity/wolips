Manipulating the Build Process

General

WOLips runs the build.xml file in your project to build the target. Please refer to the woproject documentation and/or the Ant documentation. You may alter this script to your liking.

Specific

Change the destination directory

Initially the init.build target contains this:

property name="dest.dir" value="."

This causes the build result to end up in the project root folder. You may change the property value to:

property name="dest.dir" value="${wo.localroot}${file.separator}Library${file.separator}Frameworks"

Now the build result ends up in C:\Apple\Local\Library\Frameworks (on Windows).

Alternatively you could have executed the install target instead of the build target. This would, in this case, already have the dest.dir property set correctly.

Adding web server resources

The build.woframework and/or build.woapp targets contain a section like this:

<wsresources dir=".">
<include name="addYourWebServerResourcesHere">
<exclude name="**/*.framework/**">
</wsresources>

You may, or in fact you have to, add one line per web server resource you in your project.

<wsresources dir=".">
<include name="pic1.gif">
<include name="pic2.gif">
<include name="pic3.gif">
<exclude name="**/*.framework/**">
</wsresources>
