Debugging Applications

To run or debug a program in eclipse you need to have a launch configuration for that program. There are a number of templates to create launch configurations shipped with eclipse: Java Applet, Java Application, JUnit and Run-time Workbench. With WOLips installed there is also a WOApplication template. If jMechanic was installed when you installed or last updated WOLips you will also have a Profiled WOApplication template. In theory all you need to do is select the menu option Run > Run As > WOApplication (or Run > Debug As > WOApplication) and a lauch configuration is created for you and the program launched. In real life you may have to specify a few options (and give the configuration a meaningful name). The available options are outlined below.

When you want to run/debug an application based on an already existing lauch configuration that was recently used you select Run > Debug History > 'MyConfiguration' (or Run > Run History > 'MyConfiguration').

If you want to modify an existing configuration or run/debug an application that wasn't recently used you select Run > Debug... (or Run > Run...). This is also where you create new lauch configurations when you want to make sure you get all the options and arguments correct from the beginning.

  1. Select your project, or even better the Application class
  2. Select the menu option Run > Run...(or Run > Debug...)
  3. Choose WOApplication, and click Add
  4. Search for the Application class
  5. Run/Debug

The same launch configuration is used for both running and debugging.

The NSProjectSearchPath

The launching configuration wizard generates the NSProjectSearchPath for you to enable the rapid turnaround mode for frameworks. There are two ways to expand it:

  1. In the preferences
  2. In the launch configuration wizard

Under the WOArguments tab you will find a list of default arguments including: "-NSProjectSearchPath (GeneratedByWOLips)" In the case of a launch WOLips replaces "GeneratedByWOLips" with a path to all your projects in the the workbench.(It does not matter if the projects are in folder workspace or not.) If you expand the NSProjectPath in the preferences WOLips takes care of the "," between the entires. If you expand it in the launch configuration wizard, you have to add "," between your entries and the "GenereatedByWOLips".
