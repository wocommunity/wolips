What's New

1.0.5

  • Option to set the JVMOption in the WOApplication.
  • Include/exclude patterns for the CLASSES and WOAPP_RESOURCES arrays in the PB.project.
  • Enable/disable the update of the PB.project.
  • Bugfix to prevent an update of the PB.project whenever you open or close a .java file.
  • Javadoc in the help.
  • Entry in the tips and tricks list.
  • Welcome page.
  • The WOLips perspective is the default welcome perspective.
  • Preferences page for the launch config.
  • New tab for the launch config.
  • Action to add a framework from the user home.
  • Action to add a framework from a expanded classpath variable.
  • Patterns for the incremental builder.
  • Optional ssdd target in the application build file.

1.0.4

  • This update fixes a minor problem in the incremental builder and a problem with referenced patternsets.
  • Patched org.eclipse.ant.core plugin for Eclipse 2.1.0 and 2.1.1.
  • Updated Ant tasks to support split install.

1.0.3

  • This update fixes a classpath problem in 1.0.2.

1.0.2

  • To update your projects to 1.0.2 create a new Application. Take the build.xml from the new Application and copy it into your application projects. You have to rename the project name in the copies. After that close and open all your projects once.
  • The tasks wocompile, pbindex and pbxindex(NEW) are now available within WOLips without defining there task.

  • Some minor bugfixes.

1.0.0

  • There has been changes (additions and bug fixes) related to the wobuild.properties file.< It is recommended that you rebuild it.
  • For some time there has been a memory leak in the org.eclipse.ant.core plug-in supplied with Eclipse. (The actual problem is with Ant itself). Until the problem is fixed you may want to replace the supplied plug-in with this patched version available with the WOLips download.

  • Some minor bugfixes.

0.9.8

  • Update PB.project is back in the context menu.
  • Delete WOComponent currently does nothing.
  • To use existing projects with 0.9.8 there some steps to make it work.
    1. Step 1. Go to the preferences and select rebuild wobuild.properties on next launch, and then restart Eclipse. (If you're on Windows you should then open the newly created wobuild.properties file and replace all single '\' with double '\\' - if there are any.)
    2. Step 2. Open all .classpath files and replace NEXT_ROOT with NEXT_SYSTEM_ROOT and paths to the Library dir with NEXT_LOCAL_ROOT. (Eclipse now has three class path variables related to WOLips: NEXT_ROOT, NEXT_LOCAL_ROOT and NEXT_SYSTEM_ROOT. Regardless of what NEXT_ROOT points to it cannot be used in the .classpath files.)
    3. Step 3. Modify your build files. You have to replace wo.woroot with wo.wosystemroot. Make sure that you replace all occurencies.
    4. Step 4. Have fun!
  • To profile your WOApplication download jmechanic 0.4.6. Currently the heap summary view freezes Eclipse on MacOS X.
